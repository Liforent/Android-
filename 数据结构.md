# 数组 Array

连续内存，数据类型一致，提前指定内存，不能改变

- 优点

  1. 查询速度快。O(1)
  2. 按照索引遍历方便

- 缺点

  1. 数组大小一旦固定后就无法扩容了
  2. 只能存储一个类型的数据
  3. 添加、删除的操作慢，需要移动其他元素。O(1)~O(n)。

- 常用操作的时间复杂度

  - 查询O(1)。index直接定位。

  - 插入O(1)~O(n)

    - 无序数组
      - 空间足够，直接插入末尾，O(n)
      - 空间不够，要移动数组再添加，O(n)+O(1) = O(n)
    - 有序数组
      - 空间足够，按照算法查询插入的位置
        - 二分查询
          - 查找O(log2n) + 移动O(n) = O(n)
        - 顺序查询
          - 查找O(n) + 移动O(n) = O(n)

  - 删除O(1)~O(n)

  - Java中常用的为可变数组java.util.ArrayList.里面维护了一个数组对象，==初始化容量为10==，当新增时插入元素到最后，若新增后容量超过数组容量，则进行==1.5倍扩容==，删除时找到对应元素删除，并将该元素后面所有元素向前移动一位。

    (注意Java的Util包下面没有Array类.)



# 链表 LinkedList

插入和删除快，速度O(1)，查找慢O(n). 占用内存空间多，但是可以不连续。

链表分为单链表，双向链表，循环链表，双向循环链表。

物理存储上非连续的，非顺序的存储结构，数据元素的逻辑顺序是通过链表的指针地址实现，每个元素包含数据域和指针域。

- 优点

  不需要初始化容量，可以任意加减元素，添加删除很快

- 缺点

  含有大量的指针域，占用内存大（虽然不需要连续内存），查询慢，需要遍历来查询。

适用场景：

​	数据量少，需要频繁增加删除。

- Java中的实现为java.util.==LinkedList,是双向链表==,也就是Node节点中存储了前置节点指针prev和后置节点指针next,增加了存储空间,但是可以进行头部以及尾部的插入删除,因此可以实现stack栈和queue队列以及deque双端队列的特性.

  (链表是LinkedList,而不是List,List是Java中的一个接口,继承自Collection.List是有序集合,Set是无序集合.)

# 栈 Stack

只允许在栈顶进入入栈(push)和出栈操作(pop)。（后进先出LIFO）。

栈常用一维数组或链表来实现，用数组实现的栈叫作 **顺序栈** ，用链表实现的栈叫作 **链式栈** 。

常见应用：

1. 浏览器的回退和前进
2. 检查符号是否成对出现
3. 反转字符串
4. 维护函数调用
5. 深度优先遍历DFS

栈是一种特殊的线性表，仅能在栈顶操作。特点是先进后出，常用于递归场景。

- 栈的方法

  - boolean empty() 

    测试栈是否为空

  - E peek()

    获取栈顶,不移除.

  - E push(E item)

    入栈

  - int search(Obejct o)

    返回对象在栈中的位置.

- 栈有两种实现方式,一种是数组实现,如java.util.stack,java.util.ArrayDeque,一种是链表实现,如java.util.LinkedList.用数组实现要考虑扩容缩容,用链表实现需要更多的内存空间存储节点指针.

- 栈的应用

  方法的调用，编译器平衡符号（小括号，中括号，大括号的左右平衡），HTML和XML文件中的标签匹配

# 队列 Queue

先进先出FIFO，用数组实现的队列叫作 **顺序队列** ，用链表实现的队列叫作 **链式队列** 。**队列只允许在后端（rear）进行插入操作也就是入队 enqueue，在前端（front）进行删除操作也就是出队 dequeue**

先进先出。常用于多线程队列管理。队列是一种特殊的线性表,只允许在表的前端front进行删除,在表的后端rear进行插入.

- 队列的方法

  - add(E e) 

    插入队尾,返回boolean

  - element()

    获取队头,不移除,队列为空抛出异常.返回泛型E

  - offer(E e)

    插入队尾,此方法优于add.

  - peek()

    获取头,不移除,队列为空返回null.返回泛型E

  - poll()

    获取头,并且移除,队列为空返回null.返回泛型E.

  - remove()

    获取并移除队里的头,返回泛型E

- 队列也可以用数组和链表实现.

- 队列的应用

  消息队列,打印机队列以及现实生活中先到先服务的场景.

## 队列分类

- 单队列

  存在假溢出和数组越界， `front==rear` 的时候队列为空，

- 循环队列

  可以解决假溢出和数组越界， `front==rear` 的时候队列为空或者满。

  1. 可以设置一个标志变量 `flag`,当 `front==rear` 并且 `flag=0` 的时候队列为空，当`front==rear` 并且 `flag=1` 的时候队列为满。
  2. 队列为空的时候就是 `front==rear` ，队列满的时候，我们保证数组还有一个空闲的位置，rear 就指向这个空闲位置，如下图所示，那么现在判断队列是否为满的条件就是：`(rear+1) % QueueSize==front` 。

- 双端队列Deque 

  在队列的两端都可以进行插入和删除操作的队列，相比单队列来说更加灵活。

- 优先队列 Priority Queue

  从底层结构上来讲并非线性的数据结构，它一般是由堆来实现的。

  1. 在每个元素入队时，优先队列会将新元素其插入堆中并调整堆。
  2. 在队头出队时，优先队列会返回堆顶元素并调整堆。

## 应用场景

- 阻塞队列。

  队列+阻塞操作。 队列空时，出队操作阻塞，队列满时，入队操作阻塞。使用阻塞队列可实现“生产者-消费者”模型。

- 线程池中的请求/任务队列。

- 栈。

  Stack类和Vector类在Java中已废弃，双端队列Deque可实现栈的全部功能。

- 广度优先搜索BFS。

  在图的广度优先搜索过程中，队列被用于存储待访问的节点，保证按照层次顺序遍历图的节点。



# 散链表 HashMap

一. 概述

 - 简介

   HashMap采用了数组的拉链法,存储了一组key-value.(数组+链表+红黑树,JDK1.8之后增加了红黑树部分)

 - 特点

二.相关概念

1. Hash算法

   又称摘要算法Digest,它的作用是对任意一组数据进行计算,得到一个固定长度的输出摘要.

   - Java中的hashCode()就是一种哈希算法,它可以输入任意字符串,得到固定的4个字节int整数.
   - 常用hash算法
     - MD5
     - SHA-1
     - SHA-256
     - SHA-512
     - RipeMD-160
   - hash算法的应用
     - 加密(验证数据(文件等)是否被篡改等)
     - 快速查找

2. Hash表

   Hash值的集合.

3. <a name="Hash冲突">Hash碰撞（Hash冲突）</a>

   不同的值进行Hash运算，得到同一个值，即发生了Hash碰撞。碰撞无法避免,但应该尽量减少.

   HashMap中，发生Hash碰撞后，用链表方式存储。除了单独链表法，还有开放地址法，再散列，建立公共溢出等方法来解决hash碰撞.

   - 单独链表法(链地址法,拉链法)

     Hash值相同的元素构成单链表

   - 开放地址法

     hash表中找一个空闲的位置存放发生了hash冲突的hash值.

     - 线形探查法

       依次判断hash表中的下一个单元是否为空,为空则存入

     - 平方探查法

     - 双散列函数探查法

   - 再哈希法

     - 冲突后用第二种hash算法再次进行哈希计算.冲突少了,但是增加了计算时间.

   - 建立公共溢出区等

     - hash表分为公共表和溢出表.

- put和get原理，以put为例：

  	1. 先将key，value封装到Node对象中。
  	2. 调用key的hashCode()方法，得到其hashCode值，再通过高位运算和取摸运算，来确定数组的下标。

  3. 如果该下标对应的位置上有链表了，就会拿着key和链表上每个节点的key进行equals，如果所有equals结果都是false，那么这个新的结点会被添加到链表的末尾，否则进行覆盖。

  例：hashMap.put("name","Tom").

  计算"name"的hashCode值，找到数组的下标i，再找到i对应的链表进行插入或者覆盖。

  因为是数组所以查询快，而增删都是在链表上完成，所以插入快。

- HashMap的红黑树原理

  红黑树，自平衡的二叉排序树，左右子树高度几乎一致，以此防止树退化成链表。

  jdk1.8引入了红黑树，当Hash表中单一链表的长度超过了8时，链表结构会转变成红黑树。

  链表查询慢，这样会避免当某些极端情况下，链表过长，查询效率低。

  链表查询时间复杂度O(n),红黑树查询时间复杂度O(log(n))，类似二分查询。

- 常用参数

  ``` java
  int DEFAULT_INITIAL_CAPACITY = 1 << 4 //默认初始容量是16
  int TREEIFY_THRESHOLD = 8 //默认阈值,当bucket上的链表长度大于这个数值之后链表会转为红黑树.
  int UNTREEIFY_THRESHOLD = 6
  int threshold       // 能容纳的key-value对的极限
  final float loadFactory   //加载因子，也就是数组的填充度,默认0.75,超过之后HashMap会自动扩容变为原来的2倍.
  int modCount
  int size
    
  class Node<K,V> implement Map.Entry<K,V> {
    final int hash; // 定位数组索引
    final K key;
    V value;
    Node<K,V> next;
    
    Node(...){...}
  }
  ```

  

  ​	Node[] table初始长度length默认为16，loadFactory默认为0.75， 16 * 0.75 = 12 , 当哈希桶数组长度超过了就会进行扩容，扩容后容量是之前的2倍。可以大于1  

- 扩容机制 resize （1.8做了优化）

  1. 使用新数组代替旧数组，容量 * 2

  2. 所有Node进行rehash，重新计算在数组中的索引位置

     1.8做了优化，主要是减少rehash的消耗，以及让之前hash冲突的点更分散

     问: ArrayList默认容量多少?如何扩容?

- 线程安全

  HashMap非线程安全，HashTable线程安全但是已经不用了(HashTable效率低,只是在方法上加了synchronized)，用ConcurrentHashMap。

  HashMap多线程下会出现死循环问题。

  多线程下扩容的时候可能导致Hash桶的链表由单向链表变为环形链表,从而导致get()方法死循环.

- 总结

  1. 扩容是个特别消耗性能的操作，在使用HashMap的时候，要估算map的大小避免频繁扩容
  2. 负载因子可以修改可以超过1，但是不建议轻易修改
  3. 1.8之后引入了红黑树大程度优化了hashmap性能
  4. 线程不安全

# 树 Tree

## 树的特点

1. 任意两个节点有且仅有唯一一条路径连通。
2. 一棵树如果有 n 个结点，那么它一定恰好有 n-1 条边。
3. 树不包含回路。

3. 树是N个节点的有限集合，N = 0时为空树。

   - 节点
   
     - 根节点
   
       非空树根节点为1个，空树根节点为0个。
   
     - 叶子结点
   
       没有子树的结点
   
   - 度：节点拥有的子树数目称为节点的度。
   
   - 深度：节点的层次称为树的深度或者高度。
   
   - B+树：
   
     N叉树，每个节点有多个孩子。
   
     保持数据稳定有序，插入与修改拥有较稳定的时间复杂度，自底向上插入，==mysql==数据库索引结构用的B+树。
   
   - 二叉树
   
     每个节点的度 <= 2,左右子树顺序不能颠倒，哪怕某个节点只有一个子树。
   
     二叉树顺序存储会造成空间浪费，所以考虑==链式存储==。（二叉链表）
   
     二叉树==同时有链表和数组的好处==，添加删除元素快，并且在查找方面也有很多的算法优化，二叉树在处理大批量的动态数据方面非常有用。
   
     
   
     二叉树遍历：
   
      1. 前序遍历
   
         ==父节点==->左子树->右子树
   
      2. 中序遍历
   
         左子树->父节点->右子树
   
      3. 后序遍历
   
         左子树->右子树->父节点
   
      4. 已知前序和中序，可以确定。已知中序和后序，可以确定。已知前序和后序，不能确定。（已知中序+前或者后序就能确定二叉树）
   
     二叉树种类：
   
     - 满二叉树
   
       除了叶子结点，每个节点都有2个子树
   
     - 完全二叉树
   
       假设树的深度为H，那么除了H外，其他层的的结点的子树都是2，并且第H层所有的结点都连续集中在左侧，那么这个树就是完全二叉树。
   
       堆是一种特殊的完全二叉树
   
       场景：
   
       ​	
   
     - 线索二叉树	
   
       1. 传统二叉树是用二叉链表存储的，有大量的空指针域，弦索二叉树按照一定规则修改了指针域所指向的地址，改成指向了别的节点。
       2. 普通二叉树无法知道节点的前后的位置，必须要先遍历一次。
   
       线索二叉树结构：
   
       ​	LChild，LTag，Data，RTag，RChild
   
       其中tag是boolean变量0或1，用来区分指向的是孩子还是节点。
   
     - 霍夫曼树（Huffman Tree,最优二叉树）
   
       给定N个权值作为N个叶子结点，构造一颗二叉树，若该树的带权路径长度达到最小，那么这样的树就是最优二叉树。 
   
       树的带权路径长度 = 叶结点的权值 * 叶结点到根节点的路径长度 
   
       WPL = W1 * L 1 + W2 * L2 + 。。。Wn * Ln。
   
       哈夫曼树可以用于编码，数据压缩等。
   
     ​		
   
     - 二叉排序树（二叉查找树）
   
       就是排序了的二叉树，==中序遍历后就是从小到大==
   
       无序线性表插入快，查找慢，有序线性表查找快，插入时要考虑顺序所以慢，二叉排序树整合了他们的优点。
   
        - 平衡二叉树
   
          左右子树深度之差不超过1，并且左右子树都是平衡树
   
        - 红黑树
   
          一棵自平衡的二叉排序树

# 堆 Heap

堆是一种满足以下条件的树：

堆中的每一个节点值都大于等于（或小于等于）子树中所有节点的值。或者说，任意一个节点的值都大于等于（或小于等于）所有子节点的值。

> 大家可以把堆(最大堆)理解为一个公司,这个公司很公平,谁能力强谁就当老大,不存在弱的人当老大,老大手底下的人一定不会比他强。这样有助于理解后续堆的操作。

**!!!特别提示：**

- 很多博客说堆是完全二叉树，其实并非如此，**堆不一定是完全二叉树**，只是为了方便存储和索引，我们通常用完全二叉树的形式来表示堆，事实上，广为人知的斐波那契堆和二项堆就不是完全二叉树,它们甚至都不是二叉树。
- （**二叉**）堆是一个数组，它可以被看成是一个 **近似的完全二叉树**。——《算法导论》第三版

# 布隆过滤器 Bloom Filter

## 什么是布隆过滤器？

布隆过滤器是一种**空间效率极高的概率性数据结构**，用于判断一个元素是否属于一个集合。它的特点是：

- **可能存在误判**（假阳性）：可能会错误地认为不在集合中的元素在集合中
- **不会漏判**（假阴性）：绝对不会错误地认为在集合中的元素不在集合中

## 实际应用场景

# ArrayMap

##### 一. 概述

- 前言

  HashMap为实现快速查询带来了很大的内存浪费,因此Android于2013年增加了ArrayMap类.跟SparseArray类似,ArrayMap适用于数据量较少的场景.(千级以内),如果是Int类型的,那就用SparseArray(避免了自动装箱)

- 特点

  - 区别于HashMap,ArrayMap处理[Hash冲突](#Hash冲突)的方式是==开放地址法==.
  - HashMap默认容量是16(DEFAULT_INITIAL_CAPACITY = 1 << 4),即使里边没有任何元素,而且扩容时必须是扩容成2倍.ArrayMap==默认大小为0,添加了数据后再扩容==.
  - 跟SparseArray类似,也是用的==二分法查找==,用的==两个数组==来分别存储key和value.适用于数据量==千级以内==的存储.
  - ArrayMap是专为Android设计的Map对象,使用场景比较高频,很多场景的初始数据都比较少,为了减少频繁的创建和回收
  - ==缓存了大小为4和8的两个ArrayMap==,避免频繁创建和销毁来提升性能.

# SparseArray

类似的还有SparseBooleanArray,SparseLongArray,LongSparseArray等.

- 特点

  - key只能是int类型的.(有个好处就是==避免了自动装箱==.)
  - 由==两个数组==构成,不是map集合.
  - ==查找使用==的是==二分法==.(数据量少的时候,二分发效率高.HashMap是遍历Entry),因此数据量越大效率会越低.(数据量1000以内)
  - ==不能保证插入顺序,没有Iterator.==
  - 不需要额外结构体,单个元素存储成本低.
  - ==删除数据仅标记为DELETED==,删除和插入的时候==不一定马上要移动数组==,效率高.

  总结:

  SparseArray适合key为int类型的少量数据的存储.

``` Java
public class SparseArray<E> implements Cloneable {
  // 数组中的数据如果被删除了,不会直接移动数组(提高效率),而是仅仅标记value为DELETED,也可以直接在这上边PUT来覆盖
  private static final Object DELETED = new Object();
  private boolean mGarbage = false; //是否进行垃圾回收.
  private int[] mKeys; //key必须是int类型的不可重复.比如ReyclerView的ItemType.
  private Object[] mValues;
  private int mSize;

  //默认key和value的数组长度都是10.
  public SparseArray() {
    this(10);
  }
} 
```

- 存储put

  ```Java
  public void put(int key, E value) {
    int i = ContainerHelpers.binarySearch(mKeys,mSize,key);
    // 找到了就直接覆盖value值
    if (i >= 0) {
      mValues[i] = value;
    } else {
      //根据二分找到第一个大于i的位置然后插入,之后的值往后移.
      i = ~i;
      //如果当前位置value为DELETED,那么说明可以直接用,直接覆盖就行了.
      if (i < mSize && mValues[i] == DELETED) {
        mKeys[i] = key;
        mValues[i] = value;
        return;
      }
      // mGarbage为true说明有DELETED,这时候可以通过移动,不需要扩容.
      if (mGarbage && mSize >= mKeys.length) {
        gc();
        i = ~ContainerHelpers.binarySearch(mKeys, mSize, key);
      }
      //如果位置没有Deleted,且数组已经满了,则二倍的扩容
      mKeys = GrowingArrayUtils.insert(mKeys, mSize, i, key);
      mValues = GrowingArrayUtils.insert(mValues, mSize, i, value);
      mSize++;
    }
  }
  ```

  1. 二分查找i的位置,查找到了直接替换value
  2. 如果当前i不存在,则找到第一个大于i的位置插入.如果该位置标记为DELETED,那么直接插入.否则判断需要移动还是必须扩容.
  3. 如果数组满了并且没有DELETED标记,那么只能2倍扩容后再插入了.

- gc().双指针,移动删除DELETED标记

  ```Java
  private void gc() {
    int n = mSize;
    int o = 0;
    int[] keys = mKeys;
    Object values = mValues;
    for (int i = 0; i < n; i++ ) {
      Object val = values[i
      if (val != DELETED) {	
        if (i != o) {
          keys[o] = keys[i];
          values[o] = val;
          values[i] = null;
        }
        o++;
      }
    }
  }
  ```
